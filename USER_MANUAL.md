## 1. Application Overview & Roles

### What is this application?

The Real Estate CMS is an internal platform for managing investors, documents, and communications within the fund. It covers the full investor lifecycle — from first contact through to post-investment monitoring — as well as secure document sharing with external parties.

### Roles and permissions

| Role | Description |
|------|-------------|
| **superadmin** | Full access to everything. The only role that can delete investors and manage all system settings. |
| **admin** | Nearly identical to superadmin. Can approve emails, manage document packages, capital calls, and distributions. |
| **relationship_manager** | Sees and manages only their **assigned** investors. Can create email drafts, upload documents, and create access links. |
| **operations** | Manages capital calls and distributions. Access to public and restricted documents. |
| **compliance_officer** | Access to public, restricted, and confidential documents. Can approve investors and stage changes. |
| **fund_manager** | Manages capital calls/distributions, access to all document levels. |
| **auditor** | Read-only. Access to all documents. No modifications allowed. |
| **data_room_administrator** | Manages the Data Room (upload, approve, archive documents). Access up to confidential level. |

## 2. Login & Security (2FA)

### Logging in

1. Go to: `https://[app-domain]/login`
2. Enter your **email** and **password**.
3. Click **Log In**.

### Two-Factor Authentication (2FA)

**Required for:** superadmin, admin, operations, compliance, data_room_administrator, auditor.

**First-time 2FA setup:**
1. After first login, the system automatically redirects you to `/2fa/setup`.
2. Open an authenticator app on your phone (e.g. **Google Authenticator** or **Authy**).
3. Scan the QR code shown on screen.
4. Enter the 6-digit code generated by your app.
5. Click **Enable 2FA**.

**Every subsequent login:**
1. Enter email and password.
2. The system requests a 6-digit code — open your authenticator app and enter the current code.
3. Click **Verify**.

> **Note:** The relationship_manager role does **not require** 2FA, but it is strongly recommended to set it up.

## 3. SUPERADMIN / ADMIN — Complete Guide

### 3.1 Dashboard

**URL:** `/dashboard`

The dashboard shows:

### 3.2 Investor Management

#### Viewing all investors

**URL:** `/investors`

The table shows: name, type, stage, status, assigned manager, fund.  
Click an investor's name to open their detail page.

#### Creating a new investor

**URL:** `/investors/create`

**Step by step:**

1. Click the **New Investor** button (top right on `/investors`).
2. Fill in the required fields:

| Field | Description |
|-------|-------------|
| **Investor Type** | individual / corporate / family_office / spv / fund |
| **Organization Name** | Organisation name or full name of the individual |
| **Legal Entity Name** | Legal name (can be the same as Organization Name) |
| **Jurisdiction** | Country/jurisdiction of the investor |
| **Assigned To** | Select the Relationship Manager responsible for this investor |
| **Fund** | The fund the investor is investing in (if applicable) |

3. Optionally fill in:
   - **Source of Introduction** — how you became aware of this investor
   - **Placement Agent Name/Email** — if an intermediary is involved
   - **Target Commitment Amount** — target investment amount and currency
   - **Notes** — internal notes

4. Click **Create Investor**.

> **The system automatically:** Creates the investor at stage `prospect`, status `pending`, and generates their own folders in the Data Room.

#### Investor detail page (`/investors/{id}`)

The page is divided into sections:

- **Basic Info** — core investor data
- **Eligibility** — eligibility for investment status
- **KYC** — Know Your Customer status
- **Funding** — financial data about the investment
- **Compliance** — checkboxes for compliance gates
- **Contacts** — list of contacts
- **Meetings** — logged meetings
- **Document Access Links** — links for document access
- **Email Drafts** — associated draft emails
- **Activity Log** — history of all actions

#### Editing an investor

1. Open the investor → click the **Edit** button.
2. Change the desired fields.
3. **Compliance checkboxes:** When you check a checkbox for the first time (e.g. `Board Approval`), the system automatically records the date and time.
4. Click **Update Investor**.

### 3.3 Stage Changes & Gate Checks

**URL:** `/investors/{id}/change-stage`

#### Investor stages (in order)

```
prospect
  → eligibility_confirmed
    → portal_access_granted
      → kyc_in_progress
        → kyc_completed
          → funded
            → monitored
```

#### How to change the stage:

1. On the investor detail page, click **Change Stage**.
2. The system shows a list of available stages and a **requirements checklist** for each.
3. Select the target stage.
4. The system shows what is **met** (✓) and what is **missing** (✗).
5. If all requirements are met, the **Confirm Stage Change** button becomes available.
6. Click **Confirm** — the change is recorded in the Activity Log.

#### What is checked for each stage change:

| Stage | Requirements |
|-------|--------------|
| **eligibility_confirmed** | At least one contact exists; investor_type filled in |
| **portal_access_granted** | Primary contact has an email; confirmed_professional_client ✓ |
| **kyc_in_progress** | A commitment amount exists |
| **kyc_completed** | kyc_status = completed or approved; fatca_compliant ✓ |
| **funded** | subscription_signed_date filled; board_approval_date filled; units_allotted > 0 |
| **monitored** | funded_amount > 0; welcome_letter_sent_date filled |

> **Important:** If a requirement is not met, the system **blocks** the stage change and shows what needs to be filled in. Go back to Edit and complete the missing data.

#### Viewing stage history

On the investor page, click **Activity Log** (`/investors/{id}/activity`):
- See who changed the stage and when
- All actions are logged (document downloads, compliance gate submissions, stage transitions)

### 3.4 Data Room — Uploading & Organizing Documents

**URL:** `/data-room`

#### Data Room structure overview

The Data Room is organized into **sections (folders)**:

- **Sections 1–11:** Shared folders (all investors see `public` documents)
- **Section 12:** Automatically created per investor (private folder)
- Below sections 1–11, **Investor Folders** are shown — specific folders per investor

Each folder displays: number, name, access level badge, document count.  
Click a folder to expand it and see its documents.

#### Document access levels

| Badge | Colour | Who can see it |
|-------|--------|----------------|
| **Public** | green | Everyone, including investors |
| **Restricted** | yellow | Staff only (RM, operations, admin…) |
| **Confidential** | orange | Compliance, data_room_admin, admin, superadmin |
| **Highly Confidential** | red | Admin and superadmin only |

#### Uploading one or more documents

1. Click the **Upload Document** button (top right of the Data Room page).
2. An upload modal will appear.

**Selecting a folder:**
- Use the dropdown to select the **folder** you are uploading into (e.g. "01 — Fund Overview").
- If it is an investor folder, information about that investor will be displayed.

**Drag & Drop or Browse:**
- Drag files into the drag & drop zone, **or**
- Click the zone → open File Explorer → select **one or more files** (Ctrl+click for multiple selection).

**Settings (applied to all files in the batch):**

| Field | What to enter |
|-------|---------------|
| **Version** | Document version number (default: `1.0`) |
| **Description** | Brief description of the content |
| **Access Level** | Select: Public / Restricted / Confidential / Highly Confidential |

> **The default access level is `Public`.** Change to `Restricted` only for documents that must not be visible to investors.

**Starting the upload:**
1. Click the **Upload** button.
2. The system uploads files **one by one** — a progress bar shows progress for each.
3. A ✓ appears in the queue list as each file completes.
4. The modal closes automatically after all files have been uploaded successfully.

> **Limit:** Maximum file size is **500 MB** per file.  
> **Supported formats:** PDF, DOCX, DOC, XLSX, XLS, PPTX, PPT, and others.

#### Document workflow (Status)

Every document has a status badge shown next to the access level badge:

| Status | Meaning |
|--------|---------|
| **Approved** | Visible and available for download |
| **Draft** | Awaiting submission — not yet in review |
| **Under Review** | Submitted for review, awaiting approval |
| **Archived** | Hidden, not available for download |

**Approval flow:**
1. Upload → document is automatically set to `Approved`
2. If review is needed: click **Submit** → status becomes `Under Review`
3. Admin/Superadmin sees buttons: **Approve** | **Reject** | **Archive** | **Delete**

#### Exporting the document index

Click **Export Index** (top right) → downloads an Excel file with all documents and their metadata.

### 3.5 Document Packages & Access Links

#### What is a Document Package?

A package is a **collection of documents** from the Data Room that are sent together. Each package can have **multiple access links** for different recipients.

#### Creating a Document Package

**URL:** `/document-packages/create`

1. Go to **Document Packages** in the navigation.
2. Click **Create Package**.
3. Enter:
   - **Name** — package name (e.g. "PPM Package Q2 2026")
   - **Description** — optional
   - **Notify User** — select a staff user who receives a notification when someone requests access
4. **Select documents:** Click the documents from the Data Room list that you want to include.
5. Click **Create Package**.

#### Creating an Access Link for an investor

**Scenario:** You need to send documents to an external lawyer or potential investor without them having an account.

**Step by step:**

1. Open the investor's page (`/investors/{id}`).
2. Scroll to the **Document Access Links** section.
3. Click **Create Access Link**.
4. On the form (`/document-access-links/investor/{id}/create`):
   - **Label** — a description of the purpose (e.g. "For ABC Law Firm - Due Diligence")
   - **Package** — select an existing package **OR** select individual documents to create an ad-hoc package
5. Click **Generate Link**.
6. The system generates a **unique token URL** such as: `https://[domain]/doc-access/AbCdEfGh...`
7. Click **Copy** to copy the link and send it to the recipient by email.

#### What happens when the recipient clicks the link

1. Recipient opens the link → sees a form with the package name and fields for name and email.
2. They enter their details and confirm consent (privacy notice).
3. They receive the message: "Request received — awaiting approval."
4. **Staff receives a notification** about the new request.
5. Admin/RM approves the request → recipient receives an email with a download link.
6. Recipient has **48 hours** to download the documents.

#### Managing Access Links

At `/document-access-links/investor/{id}`:
- View all links for the investor
- Number of pending requests per link
- Delete a link (revoke access)

#### Approving access requests

**URL:** `/document-access-requests`

1. Go to **Access Requests** in the navigation.
2. The table shows: name, email, request date, which link, status.
3. Click **Approve** — the recipient receives an email with a download link (48h window).
4. Click **Reject** — the recipient receives a rejection email.

### 3.6 Composing & Sending Emails

> **Important:** There is no "direct send" button in this application. All emails go through the **Email Drafts** workflow: compose → save → approve → send. This applies to both Admin and Relationship Manager roles.

#### Composing an email (Compose Email)

**Access:** From any investor's detail page → **Compose Email** button, or via **Email Drafts → Create Draft** in the navigation.

**Step by step:**

1. Open an investor's detail page and click **Compose Email**.
2. Fill in the form:
   - **On Behalf Of** — select the person whose name appears as sender (optional)
   - **Subject** — email subject line
   - **Load Body Template** — optionally load a pre-written template as a starting point
   - **Email Body** — write or edit the content (rich text editor)
   - **Signature** — select a signature block (optional)
   - **CC** — optional CC email; if the investor has a placement agent, a checkbox appears to CC them automatically
   - **Attach Documents** — select documents from the Data Room to attach
3. Choose an action:
   - **Save as Draft** — saves the email, does not submit for approval yet
   - **👁 Save & Preview** — saves as draft and immediately opens a preview of how the email will look to the investor
   - **Submit for Approval** — saves and sends to admin for review

#### Bulk email (multiple investors at once)

**URL:** `/investors/send-email/bulk` — accessible via **Investors → Bulk Send Email**

1. Use the filters to select recipients:
   - **Send To** — All Investors or By Stage
   - **Stage** — appears when "By Stage" is selected; select the specific stage
   - **Only my assigned investors** — checkbox to limit to investors assigned to you
2. The recipient list updates automatically showing names, emails, and stages.
3. Select a **template** from the list below.
4. Optionally add document attachments.
5. Click **Send Email** — emails are sent to all listed recipients.

> **Note:** Bulk email sends immediately without an approval step.

### 3.7 Email Drafts — Approval Workflow

#### How the workflow works

```
RM composes email → Submits for Approval → Admin reviews → Approves → RM sends
```

Only the **creator** of a draft can send it once approved. Admin approves, but does not send.

#### Viewing pending drafts

**URL:** `/email-drafts`

Admin/Superadmin sees a dedicated **Pending Approval** section at the top of the page.

#### Approving a draft

1. Click **Review & Edit** on a draft in the Pending Approval table.
2. Review the content; click **Preview** to see how the email will render for the investor.
3. If correct: click **Approve**.
4. The creator receives access to the **Send** button — only they can trigger sending.

#### Rejecting / returning a draft

1. On the preview page, click **Reject** (or edit the draft and change the content).
2. The draft reverts to `draft` status — the creator must resubmit it.

### 3.9 Distributions

**URL:** `/distributions`

#### Creating a Distribution

1. Click **Create Distribution**.
2. Fill in:
   - **Title** — name (e.g. "Q1 2026 Dividend Distribution")
   - **Type** — dividend / return_of_capital / profit_share / other
   - **Total Amount** — total distribution amount

4. Click **Create** → status is `Draft`.

```
Draft → [Approve] → Approved → [Process] → Processing → Completed
```

1. Open the Distribution → click **Approve**.
2. Click **Process** when you begin processing payments.
3. Mark individual payments as **Paid** as they are received.

### 3.10 Payments

**On the detail page of a Capital Call or Distribution:**

#### Marking a payment as paid

1. Find the investor in the table.
2. Click **Mark as Paid**.
3. Enter:
   - **Payment Method** — wire_transfer / ach / check / cash
   - **Reference Number** — transaction reference number
   - **Paid Date** — date (defaults to today)
4. Click **Confirm**.
5. The system automatically updates the totals and the Capital Call/Distribution status.

#### Bulk mark as paid

For multiple investors at once:
1. Check the checkboxes next to the investors.
2. Click **Mark Selected as Paid**.
3. Enter the method and reference number to apply to all.

#### Reversing a payment

If you marked a payment as paid by mistake:
1. Find the payment → click **Reverse**.
2. The payment reverts to `Pending` status.
3. Totals are automatically recalculated.

### 3.14 Investor Portal Access

#### Granting portal access to an investor

1. Open the investor's page.
2. Verify that a **primary contact** with an email address exists (required!).
3. Verify the stage is at least `portal_access_granted`.
4. Click **Create Portal Access**.
5. The system:
   - Creates an InvestorUser account (email + auto-generated password)
   - Sends a login credentials email to the investor
   - Redirects to `/investor/login`

> **Note:** The investor must pass through the **Compliance Gate** on first login (confirming professional client status, confidentiality, and PPM) before they can access documents.

#### What investors see in the portal

- **Dashboard:** commitment amount, funded %, capital call payments, distributions
- **Profile:** their own data (read-only)
- **Documents:** folders they have access to, only documents with access level = `Public`

### 4.8 Creating Document Packages & Access Links

**Scenario:** A prospective investor wants to review certain fund documents. You need to send them a secure link that does not require creating an account.

#### Step 1: Create a Package (if one doesn't exist)

1. Go to `/document-packages/create`.
2. Enter the package name (e.g. "Initial DD Package — [Investor Name]").
3. Select the documents to include.
4. Click **Create Package**.

#### Step 2: Create an Access Link

1. Open the investor's page.
2. **Document Access Links** section → click **Create Access Link**.
3. Select:
   - **Label** — e.g. "Due Diligence for XYZ Advisors"
   - **Package** — select the package you just created
4. Click **Generate Link**.
5. Click **Copy Link** and save the link.

#### Step 3: Send the link to the recipient

Send the link (e.g. by email): `https://[domain]/doc-access/[token]`

#### Step 4: Monitor requests

1. The recipient clicks the link → enters name + email → waits for approval.
2. You receive a notification about the new request.
3. Go to `/document-access-requests`.
4. Click **Approve** — the recipient receives an email with a download link.

### 4.9 Composing & Sending Emails

> **Important:** There is no direct "Send" button for individual emails. All emails must go through the approval workflow: compose → submit for approval → admin approves → you send.

#### Step by step — composing an email

   - **On Behalf Of** — whose name appears as sender (optional)

   - **Signature** — select a signature block

   - **Save as Draft** — save without submitting

- The draft appears in the admin's **Pending Approval** queue
- Admin reviews and approves (or rejects with comments)
- Once approved, you see a **Send** button on the draft — click it to send
- The email goes to the investor's **primary contact** email address
- It is automatically archived in the Data Room → Section 12 (Communication Log)

## 6. Public Document Access (No Login Required)

This is the mechanism for sharing documents with people who **do not have an account** in the system.

### Flow for an external party:

1. **Receives a link** (e.g. from an RM by email): `https://[domain]/doc-access/[token]`
2. **Opens the link** → sees the document package name and a form
3. **Fills in:**
   - Full name
   - Email address
   - Confirms consent to the Privacy Notice
4. Receives the message: **"Your request has been received. Awaiting approval."**
5. Once approved by admin/RM, receives an **email with a download link**
6. The link is active for **48 hours** from the time of approval
7. Can download all documents in the package

> **Consent tracking:** The system automatically records the date, IP address, and Privacy Notice version — for compliance purposes.
